# Release Notes

Read the release range and group user-visible changes by outcome. Separate breaking changes, fixes, and operational notes. Do not infer features that are absent from the diff.

Follow the project voice and formatting in [the style guide](references/style-guide.md). Cite pull requests or commits when the repository provides them.
